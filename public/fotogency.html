Group Project: Fotogency Case Study

Project Overview:

Fotogency is a collaborative web project developed as part of a group effort to practice front-end skills. The project was based on cloning an existing professional photography agency website to learn layout, structure, and design principles. My primary contribution was developing the Home tab using semantic HTML5.

Objectives:

  • Practice front-end development by cloning a professional photography website.
  • Develop the Home tab using semantic HTML5.
  • Collaborate effectively with team members via GitHub.
  • Learn project structure, version control, and team communication.

Tech Stack:

  • HTML5
  • CSS3 (Flexbox & Grid)
  • JavaScript (for interactivity in other sections)
  • GitHub (Repository for collaboration)
  • Netlify (Deployment)

Features (Home Tab):

  • Hero section introducing the agency.
  • Clean typography and color scheme aligned with photography theme.

My Contribution:

  • Developed the Home tab using semantic HTML5 structure.
  • Created the GitHub repository to manage project files and team collaboration.
  • Started adding team members and organizing the workflow via Git.
  • Ensured the Home page replicated the design of the reference website accurately.

Challenges & Solutions:

  • Coordinating with team members → Used GitHub repo for smooth communication and file management.
  • Learning by cloning → Carefully analyzed the original site structure to understand layout and styling.
  • No responsiveness → Focused purely on layout and HTML/CSS structure for skill development.

Outcome:

  • Successful practice project that enhanced front-end skills.
  • Developed a functional Home tab using semantic HTML5 and CSS3.
  • Learned GitHub collaboration, repository management, and team workflow.
  • Deployed version accessible at: Fotogency Website
