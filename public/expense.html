Expense Tracker Case Study

Project Overview:

The Expense Tracker is a full-stack application designed to help users manage their daily expenses and track financial habits. Users can add, update, delete, and categorize expenses.

Objectives:

  • Allow users to record and monitor expenses.
  • Provide categorized expense management with CRUD operations.
  • Offer simple insights into spending habits.
  • Ensure a clean and intuitive design.

Tech Stack:

  • Java
  • Spring Boot
  • React.js
  • MySQL
  • HTML5, CSS3

Features:

  • Expense Management (Add/Edit/Delete)
  • Categorization of expenses

Functionality Flow:

  • Frontend (React.js): Users manage expenses and categories.
  • Backend (Spring Boot): REST APIs handle CRUD operations and basic calculations.
  • Database (MySQL): Stores categories and expense records.

Deployment:

  • Frontend deployed on Netlify.
  • Backend deployed on Railway.
  • Full CI/CD pipeline ensures smooth testing, building, and deployment.

Challenges & Solutions:

  • Data management → Ensured accurate CRUD operations and validation.
  • User experience → Kept the interface simple and easy to navigate.
  • Deployment → Set up CI/CD for automated updates and consistent live version.

Outcome:

  • Functional personal finance tracking tool.
  • Easy-to-use interface for managing expenses and categories.
  • Deployed and ready for future enhancements like income tracking and budgeting.
  • Complete SDLC process from development → testing → deployment → monitoring.
